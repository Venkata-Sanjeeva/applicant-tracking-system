Applicant Tracing System API Documentation

This document describes the endpoints and functionality of the Applicant Tracing System API, which allows users to manage application data (create, read, update, and delete).

Base URL

http://localhost:3000

Endpoints

1. Upload Application Data

Endpoint:

POST /api/upload

Description: Uploads a new application data object to the database.

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "field1": "value1",
  "field2": "value2",
  "..."
}

Response (201 - Created):

{
  "dataObj": {
    "_id": "unique_id",
    "field1": "value1",
    "field2": "value2",
    "...": "...",
    "createdAt": "timestamp",
    "updatedAt": "timestamp"
  }
}

2. Get All Application Data

Endpoint:

GET /api/getData

Description: Retrieves all application data stored in the database.

Response (200 - OK):

{
  "applications": [
    {
      "_id": "unique_id",
      "field1": "value1",
      "field2": "value2",
      "..."
    },
    {
      "_id": "unique_id2",
      "field1": "value1",
      "field2": "value2",
      "..."
    }
  ]
}

3. Update Application Data

Endpoint:

PATCH /api/update

Description: Updates an existing application data object in the database by its ID.

Query Parameters:

?id=application_id

Request Headers:

{
  "Content-Type": "application/json"
}

Request Body:

{
  "field1": "new_value1",
  "field2": "new_value2",
  "..."
}

Response (200 - OK):

{
  "msg": "Application Updated Successfully!"
}

Response (400 - Bad Request):

{
  "msg": "Application not found!!!"
}

4. Delete Application Data

Endpoint:

DELETE /api/delete

Description: Deletes an application data object from the database by its ID.

Query Parameters:

?id=application_id

Response (200 - OK):

{
  "msg": "Deleted Successfully!"
}

Response (400 - Bad Request):

{
  "msg": "Application not found!!!"
}

Error Responses

500 Internal Server Error:

{
  "msg": "Error deleting application"
}

Setup Instructions

  1. Install dependencies:
    2. npm install express mongoose dotenv cors
  2. Start the server:
    3. node server.js
  3. Set your .env file with the following:
    4. MONGO_URI=your_mongo_db_connection_string
